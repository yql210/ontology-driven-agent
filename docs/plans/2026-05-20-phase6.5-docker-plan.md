## Task 1: Dockerfile（多阶段构建）

**文件**: `Dockerfile`（新文件）

```dockerfile
# Stage 1: Build
FROM python:3.13-slim AS builder
COPY --from=ghcr.io/astral-sh/uv:latest /uv /bin/uv
WORKDIR /app
COPY pyproject.toml uv.lock ./
RUN uv sync --frozen --no-dev --no-install-project
COPY src/ src/
RUN uv sync --frozen --no-dev

# Stage 2: Runtime
FROM python:3.13-slim
WORKDIR /app
COPY --from=builder /app/.venv /app/.venv
COPY src/ /app/src/
COPY pyproject.toml /app/
ENV PATH="/app/.venv/bin:$PATH"
ENV PYTHONUNBUFFERED=1
ENTRYPOINT ["layerkg"]
```

**测试**: `docker build -t layerkg .` 成功

## Task 2: docker-compose.yml（编排 Neo4j + LayerKG）

**文件**: `docker-compose.yml`（新文件）

- `neo4j` 服务：官方 neo4j:5 image，7687/7474 端口
- `chromadb` 服务：官方 chromadb image，8000 端口
- `layerkg` 服务：build 自 Dockerfile，depends_on neo4j
- `.env` 环境变量注入
- volumes: neo4j_data, chroma_data 持久化

**关键配置**:
```yaml
services:
  neo4j:
    image: neo4j:5
    environment:
      NEO4J_AUTH: neo4j/${NEO4J_PASSWORD:-REDACTED_PASSWORD}
    ports: ["7687:7687", "7474:7474"]
    volumes: [neo4j_data:/data]

  chromadb:
    image: chromadb/chroma
    ports: ["8000:8000"]
    volumes: [chroma_data:/chroma/chroma]

  layerkg:
    build: .
    environment:
      NEO4J_URI: bolt://neo4j:7687
      NEO4J_USER: neo4j
      NEO4J_PASSWORD: ${NEO4J_PASSWORD:-REDACTED_PASSWORD}
      CHROMA_PERSIST_DIR: /data/chroma
    depends_on: [neo4j, chromadb]
    volumes: [./data:/data]
    profiles: ["cli"]  # 默认不启动，按需 docker compose run

volumes:
  neo4j_data:
  chroma_data:
```
