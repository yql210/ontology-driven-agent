# Phase 6.4 设计方案：本体演化（Ontology Evolution）

**日期**: 2026-05-20
**前置条件**: Phase 6.1-6.3 完成 (1089 tests, commit 0bb3296)
**目标**: Schema 变更不破坏现有数据
**审核**: Claude Code V1 9/10, V2 6/10 → 综合修改后版本

---

## 1. 问题陈述

当前 LayerKG 的 schema（6实体+11关系+约束规则）硬编码在 Python 代码中。Schema 变更只能通过代码部署实现，**数据库中的存量数据没有版本标记**，无法：

1. 知道当前数据库对应哪个版本的 schema
2. 在 schema 变更后自动迁移存量数据
3. 保证后向兼容查询

## 2. 解决方案

### 核心思路：轻量级 Schema 版本追踪 + 声明式迁移

```
SchemaVersion v1.0.0 (当前) → v1.1.0 (未来) → v2.0.0 (破坏性) → ...
         ↓                        ↓                      ↓
   Migration 注册             Migration 链            Migration 链
         ↓                        ↓                      ↓
   Neo4j 版本节点             数据迁移               数据迁移
```

### 2.1 SchemaVersion 常量 + 注册表 + 状态枚举

新文件: `src/layerkg/schema_version.py`

```python
from enum import Enum
from dataclasses import dataclass

CURRENT_SCHEMA_VERSION = "1.0.0"

class SchemaStatus(Enum):
    EMPTY = "empty"           # 无 SchemaVersion 节点（空库）
    MATCH = "match"           # 版本匹配
    BEHIND = "behind"         # DB 版本低于代码（需要迁移）
    AHEAD = "ahead"           # DB 版本高于代码（旧代码连新库）

@dataclass
class SchemaVersionInfo:
    version: str           # "1.0.0"
    description: str       # "初始本体：6实体11关系"
    applied_at: str        # ISO 8601 timestamp

def register_schema_version(store: GraphStore) -> None:
    """在 Neo4j 创建/更新 SchemaVersion 节点。
    使用 MERGE 保证幂等。"""

def get_current_db_version(store: GraphStore) -> str | None:
    """查询 Neo4j 中最新 SchemaVersion 节点的 version。
    MATCH (sv:SchemaVersion) RETURN sv.version ORDER BY sv.applied_at DESC LIMIT 1"""

def check_schema_version(store: GraphStore) -> SchemaStatus:
    """检查 DB 版本与 CURRENT_SCHEMA_VERSION 的关系，返回 SchemaStatus 枚举。"""
```

### 2.2 迁移框架

新包: `src/layerkg/migrations/`

```
src/layerkg/migrations/
├── __init__.py       # MigrationBase 抽象类
├── registry.py       # MigrationRegistry: 注册+排序所有迁移
└── runner.py         # MigrationRunner: 执行迁移链（含并发安全+原子性）
```

**MigrationBase 抽象类** (`src/layerkg/migrations/__init__.py`):

```python
from abc import ABC, abstractmethod

class MigrationBase(ABC):
    """迁移基类。所有迁移必须实现 upgrade 和 downgrade。
    
    约束：
    - upgrade 必须幂等（可安全重复执行）
    - downgrade 应尽可能恢复到迁移前状态
    """
    version_from: str     # 源版本，"0.0.0" 表示初始（无版本）
    version_to: str       # 目标版本
    description: str

    @abstractmethod
    def upgrade(self, store: GraphStore) -> None: ...

    @abstractmethod
    def downgrade(self, store: GraphStore) -> None: ...
```

**MigrationRegistry** (`src/layerkg/migrations/registry.py`):

```python
class MigrationRegistry:
    """迁移注册表。按版本顺序维护所有迁移。"""
    _migrations: list[MigrationBase]

    def register(self, migration: MigrationBase) -> None: ...
    def get_migration_path(self, from_ver: str, to_ver: str) -> list[MigrationBase]: ...
    def get_latest_version(self) -> str: ...
```

**MigrationRunner** (`src/layerkg/migrations/runner.py`):

```python
import fcntl
from pathlib import Path

class MigrationRunner:
    """迁移执行器。包含并发安全和失败恢复。"""
    
    def __init__(self, store: GraphStore, registry: MigrationRegistry): ...

    def _acquire_lock(self):
        """文件锁防止并发迁移。单机部署场景足够。"""
        lock_path = Path.home() / ".layerkg" / "migrate.lock"
        lock_path.parent.mkdir(parents=True, exist_ok=True)
        lock_file = open(lock_path, "w")  # noqa: SIM115
        fcntl.flock(lock_file.fileno(), fcntl.LOCK_EX | fcntl.LOCK_NB)
        return lock_file

    def _release_lock(self, lock_file) -> None:
        fcntl.flock(lock_file.fileno(), fcntl.LOCK_UN)
        lock_file.close()

    def run_pending(self) -> list[str]:
        """执行所有待执行的迁移。每个迁移失败时回滚该步骤并抛异常。
        
        原子性保证：Neo4j session.run() 是 auto-commit transaction，
        每个 Cypher 语句自动提交。迁移步骤内的多个操作通过
        execute_write 包装为单个事务。
        """
        lock = self._acquire_lock()
        try:
            status = check_schema_version(self.store)
            if status == SchemaStatus.MATCH:
                return []
            if status == SchemaStatus.AHEAD:
                raise SchemaMigrationError(
                    f"DB version ahead of code: {get_current_db_version(self.store)} > {CURRENT_SCHEMA_VERSION}"
                )
            # status == EMPTY or BEHIND
            current = get_current_db_version(self.store) or "0.0.0"
            target = CURRENT_SCHEMA_VERSION
            path = self.registry.get_migration_path(current, target)
            applied = []
            for migration in path:
                try:
                    migration.upgrade(self.store)
                    register_schema_version(self.store)
                    applied.append(migration.version_to)
                except Exception as e:
                    # 单步失败：记录已应用的，抛出异常
                    logger.error(f"Migration {migration.version_to} failed: {e}")
                    raise SchemaMigrationError(
                        f"Migration {migration.version_from} → {migration.version_to} failed: {e}. "
                        f"Previously applied: {applied}"
                    ) from e
            return applied
        finally:
            self._release_lock(lock)

    def rollback(self, to_version: str) -> list[str]:
        """回滚到指定版本。"""
        lock = self._acquire_lock()
        try:
            current = get_current_db_version(self.store) or "0.0.0"
            # 构建反向迁移路径
            path = list(reversed(self.registry.get_migration_path(to_version, current)))
            rolled_back = []
            for migration in path:
                migration.downgrade(self.store)
                register_schema_version(self.store)
                rolled_back.append(migration.version_from)
            return rolled_back
        finally:
            self._release_lock(lock)
```

### 2.3 集成点

#### neo4j_store.py — ensure_constraints() 扩展

在现有 `ensure_constraints()` 末尾增加：
```python
# 新增：为 SchemaVersion 创建 UNIQUE 约束
session.run("CREATE CONSTRAINT IF NOT EXISTS FOR (n:SchemaVersion) REQUIRE n.version IS UNIQUE")
# 注册当前版本
register_schema_version(self)
```

#### builder.py — build() 入口增加版本检查

```python
# build() 方法开头增加：
status = check_schema_version(self.graph_store)
if status == SchemaStatus.BEHIND or status == SchemaStatus.EMPTY:
    runner = MigrationRunner(self.graph_store, get_default_registry())
    applied = runner.run_pending()
    if applied:
        logger.info(f"Auto-applied {len(applied)} schema migrations")
elif status == SchemaStatus.AHEAD:
    raise LayerKGError(
        f"Database schema ({get_current_db_version(self.graph_store)}) "
        f"is ahead of code ({CURRENT_SCHEMA_VERSION}). Please update LayerKG."
    )
```

#### cli.py — 新增 migrate 命令

```python
@cli.command()
@click.option("--target", default=None, help="Target version (default: latest)")
def migrate(target):
    """Run pending schema migrations."""
    store = Neo4jStore.from_config(LayerKGConfig.from_env())
    registry = get_default_registry()
    runner = MigrationRunner(store, registry)
    if target:
        applied = runner.rollback(target)
        click.echo(f"Rolled back to {target}: {applied}")
    else:
        applied = runner.run_pending()
        click.echo(f"Applied {len(applied)} migrations: {applied}")
```

## 3. Scope 边界（不改什么）

| 不改 | 理由 |
|------|------|
| 现有 6 实体 11 关系定义 | 这是 v1.0.0 的 schema，不应改变 |
| `validate_relation_constraint()` 逻辑 | 约束校验已正确 |
| 查询接口 (`query`, `get_relations`) | 后向兼容通过迁移保证，而非查询时适配 |
| 不做 AI 自动 schema 演化 | 未来 Butler 的职责 |
| Butler/Web 集成 | P2，后续迭代 |

## 4. 关键设计决策

| 决策 | 理由 |
|------|------|
| 版本号用 `1.0.0` 语义化版本 | 当前 schema 已稳定 |
| 文件锁（`fcntl.flock`）而非分布式锁 | 项目单机部署，文件锁足够 |
| 每个迁移失败时停住并报告已应用版本 | 不自动回滚（回滚可能引入更多问题），提供 `rollback()` 方法手动回滚 |
| `SchemaStatus` 枚举替代 `bool` | 区分空库/匹配/落后/领先四种状态 |
| `SchemaVersion` UNIQUE 约束 | 防止重复版本节点 |
| 迁移目录放在 `src/layerkg/migrations/` | 作为 Python 包管理，方便 import |
| v1.0.0 迁移脚本是空操作 | 只注册版本节点，不改变数据 |
